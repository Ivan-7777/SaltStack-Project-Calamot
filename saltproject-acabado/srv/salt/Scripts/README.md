# Salt service test scripts

Run these scripts from the Salt master.

Main entry points:

- `test_all.sh`: runs all checks.
- `test_selected_services.py`: checks only services from the current generated `service_mapping`.
- `test_firewall_dns.sh`: DNS and firewall policy checks.
- `test_dhcp.sh`: DHCP server and relay checks.
- `test_wordpress.sh`: Apache/PHP/WordPress checks.
- `test_zabbix.sh`: frontend, server, registered hosts and agent polling.
- `test_restic.sh`: restic repository/snapshots. Set `RUN_BACKUP=1` to run backup scripts.
- `test_wireguard.sh`: WireGuard service and interface checks.
- `wireguard-cliente.sh`: existing helper to generate a WireGuard client config.
- `wirecliente.sh`: existing simple WireGuard client helper.

Examples:

```bash
/srv/salt/Scripts/test_all.sh
/srv/salt/Scripts/test_zabbix.sh
RUN_BACKUP=1 /srv/salt/Scripts/test_restic.sh
```

The scripts intentionally read the active pillar generated by the form, so they
only validate services selected for the current deployment.
